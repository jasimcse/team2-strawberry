Описание на директориите и файловете:
 \
 +-+- src                    = съхранява изходен код на Java класове
 | +--- META-INF             = настройки на persistance интерфейсите, не мисля да ги използваме, но все пак не ги пипай
 | +--- log4j.properties     = не съм сигурен, но не пипай :)
 | +--- com.sun.faces.config = hack, за да накараме google hosting-а да работи с JSF, не пипай
 | +--- org.jboss.el.util    = hack, за да накараме google да работи с Unified Expression Language 2.0 + parameterized method invocation ~ Unified Еxpression Language 2.2 
 | +--- ...                  = същинска структура на класовете на нашия проект
 | 
 +-+- war                    = съхранява всичко което се разполага на сървъра
 | +--- resources            = директория, която съхранява статични файлове (не генерирани динамично)
 | |                           например CSS, PNG, JPG файлове или статични HTML документи
 | +--- META-INF             = не знам :)
 | |
 | +-+- WEB-INF              = настройки на сървъра, не пипай
 | | +-+- appengine-generated = май за сега съхранява само данните на базата данни на локалния сървър
 | | | +--- datastore-indexes-auto.xml = автоматично генерирани дефиниции на индексите, използвани за извършването на заявки
 | | | +--- local_db.bin     = данни на базата данни; изтрий, ако искаш да си зачистиш напълно хранилището
 | | |
 | | +--- lib                = runtime библиотеки на сървъра, не пипай
 | | +--- wsdl               = дефиниции на уеб услугите предлагани от системата
 | | +--- soap               = вътре стоят файлове, указващи къде да се пращат съобщенията за уеб услугите
 | | +--- classes            = компилираните класове на приложението, не commit-вай нищо вътре
 | | +--- appengine-web.xml  = настройки на google hosting-а
 | | +--- faces-config.xml   = настройки на JSF
 | | +--- logging.properties = настройки на logger-а на google hosting-а
 | | +--- web.xml            = настройки на servlet container-а, не пипай
 | |
 | +--- ....xhtml            = файлове които ще се преобразуват в JSF файлове
 |
 +- models                   = съхранява моделите на разраотката, EA файла например
 +- docs                     = съхранява документите на разработката
 |
 +- wiki                     = съхранява wiki страниците 
 |
 +- branches                 = системна папка на SVN, мисля да не я използваме
 +- tags                     = системна папка на SVN
 +- trunk                    = системна папка на SVN, не знам дали ще я използваме
 |
 +- runwsgen.bat             = скрипт за генериране на сорс файловете, класовете и WSDL файловете за уеб услугите 
 |
 +- .settings                = настройки на Eclipse, май трябва да се игнорира
 +- .metadata                = настройки на Eclipse, май трябва да се игнорира
 +- .classpath               = път до класовете, файл на Eclipse
 +- .project                 = настройки на Eclipse
